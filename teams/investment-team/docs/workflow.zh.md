# 工作流程

从信号检测到决策执行的完整投资工作流程。该流程专为智能体自主运行设计，在关键决策节点保留人类监督。

---

## 概述

```
信号检测 → 信号分级 → 研究/建模 → 审计 → 决策 → 执行 → 反馈闭环
```

以下逐步展开说明。

---

## 1. 信号检测

**执行者：** 投资协调员
**位置：** 外部数据源、`#intel`

协调员持续监控以下数据源，捕捉投资相关信号：
- 链上数据（巨鲸异动、TVL 变化、代币流向）
- 市场数据（价格行情、成交量异动、资金费率）
- 新闻推送（监管公告、合作伙伴关系、安全事件）
- 社交媒体（舆情变化、KOL 动态）
- 安全信息（漏洞披露、攻击事件报告）

**示例：**
> 检测到信号：ETH 在 3 家主要交易所出现资金费率倒挂，4 小时内成交量激增 340%。

---

## 2. 信号分级

**执行者：** 投资协调员
**位置：** `#intel`、`#general`

协调员按照红/黄/绿框架对每个信号进行分级：

| 级别 | 颜色 | 判定标准 | 处理方式 |
|------|------|---------|---------|
| 紧急 | 红色 | 黑天鹅事件、价格波动超过 10%、安全漏洞 | 立即上报战略决策人 |
| 重要 | 黄色 | 业绩不及预期、巨鲸异动、监管动态 | 纳入每日报告 |
| 常规 | 绿色 | 市场数据更新、周期性信息 | 自动归档 |

**红色信号处理流程：**
```
信号检测
      │
      ▼
协调员判定为红色
      │
      ├──→ 广播至 #general（所有智能体收到警报）
      │
      └──→ 立即私信战略决策人，附带：
           - 信号摘要
           - 严重程度评估
           - 建议操作
           - 时间敏感度
```

**黄色/绿色信号处理流程：**
```
信号检测
      │
      ▼
协调员判定为黄色或绿色
      │
      ├──→ 黄色：加入每日报告队列
      │
      └──→ 绿色：带时间戳自动归档
```

## 3. 每日报告

**执行者：** 投资协调员
**位置：** `#general`
**时间：** 10:00 和 22:00（本地时间）

协调员每日产出两份报告，内容涵盖：
- 上次报告以来收到的所有信号汇总
- 黄色信号分析和背景说明
- 进行中的研究任务状态
- 模型产出要点
- 待战略决策人处理的决策事项
- 安全审计员的审计发现（如有）

**每日报告结构示例：**
```
=== 每日报告（10:00）===

## 信号汇总（过去 12 小时）
- 红色：0
- 黄色：3
- 绿色：14

## 黄色信号
1. [黄色] Uniswap 治理提案 #47 — 潜在手续费开关
   - 影响：中等（影响 UNI 代币价值论点）
   - 处理：已向研究分析师分派研究任务

2. [黄色] 巨鲸钱包 0x3f... 转移 12,000 ETH 至交易所
   - 影响：中低（潜在抛压）
   - 处理：持续监控中

3. [黄色] SEC 委员就 DeFi 监管发表讲话
   - 影响：中等（监管不确定性）
   - 处理：全文分析进行中

## 进行中的研究
- 研究分析师：Uniswap 手续费开关尽调 — 预计 6 小时完成
- 投资工程师：投资组合风险模型更新 — 已完成，结果见 #research

## 待决事项
- 当前无需战略决策人立即处理的事项

## 审计
- 安全审计员 03:30 UTC 审计：通过 — 无发现
```

## 4. 研究分派

**执行者：** 投资协调员 → 研究分析师
**位置：** 私信（协调员 → 研究分析师）、`#research`

当信号需要深入调查时，协调员分派研究任务：

1. 协调员通过私信向研究分析师发送结构化任务：
   - 研究主题：调查什么
   - 研究范围：需要回答的具体问题
   - 截止时间：预期完成时间
   - 优先级：紧急程度
   - 背景信息：相关信号、可参考的历史研究

2. 研究分析师确认收到并开始研究

3. 研究分析师将完成的尽调报告发布到 `#research`

**分派示例：**
```
收件人：研究分析师
任务：Uniswap 手续费开关提案尽职调查
范围：
  - 收入影响建模（现行 vs 拟议费率结构）
  - 治理投票可能性（法定人数分析、代表立场）
  - 历史先例（其他协议手续费开关案例）
  - 风险评估
截止时间：12 小时
优先级：中
背景：来自治理追踪器的黄色信号。UNI 占投资组合 4%。
```

## 5. 研究执行

**执行者：** 研究分析师
**位置：** `#research`

研究分析师按照结构化流程进行尽职调查：

1. **数据收集** — 收集链上数据、项目文档、治理论坛、市场数据
2. **深度分析** — 评估团队、技术、代币经济学、市场定位、风险
3. **历史回测** — 在适用的情况下，对类似事件/策略进行历史回测
4. **报告撰写** — 产出结构化尽调报告，明确结论和风险评估
5. **交付** — 发布到 `#research` 并通知协调员

**质量门控：** 每份尽调报告必须包含：
- [ ] 明确的论点声明（看多/看空/中性，附理由）
- [ ] 量化风险评估（低/中/高，附支撑数据）
- [ ] 历史背景或回测结果
- [ ] 已识别的未知因素和数据缺口

## 6. 投资建模

**执行者：** 投资工程师
**位置：** `#research`

投资工程师与研究工作并行，构建和维护投资模型：

1. **模型构建** — 构建量化模型（定价模型、风险模型、组合优化）
2. **策略回测** — 用历史数据测试投资策略
3. **平台开发** — 构建和维护数据管道、仪表盘、分析工具
4. **模型更新** — 融入最新研究数据和市场行情
5. **产出交付** — 将模型结果发布到 `#research`

**模型产出示例：**
```
=== 投资组合风险模型更新 ===
日期：2026-02-26
模型版本：v2.3

投资组合 VaR（95%，24h）：-4.2%
最大回撤（30 天）：-18.7%
相关性矩阵：已更新（BTC-ETH：0.82，BTC-SOL：0.71）
集中度风险：高 — 前 3 大持仓占比 67%

建议：降低集中度。考虑将前 3 大持仓占比降至 55% 以下。
```

## 7. 安全审计

**执行者：** 安全审计员
**位置：** `#intel`、私信给战略决策人和协调员

安全审计员执行持续性和定期安全审计：

### 定期审计（每日 03:30 UTC）

全面审查，覆盖以下方面：
- 投资逻辑审计：当前持仓是否与既定投资论点一致？
- 合规检查：风险限额是否得到遵守？
- 系统性风险扫描：投资组合的相关性、集中度、流动性风险
- 平台安全：扫描团队使用的外部平台和工具的漏洞（如 DeFi 协议）
- 流程审计：团队工作流程是否得到执行？

### 持续监控

- 标记违反风险政策的投资决策
- 对使用中的平台安全漏洞发出警报
- 识别流程偏差（报告缺失、研究逾期）

### 双线报告

所有发现同时报告给战略决策人和协调员：
```
审计发现
      │
      ├──→ 私信战略决策人（战略层面知悉）
      │
      └──→ 私信协调员 / #intel（运营层面行动）
```

**质量门控：** 每次审计必须产出：
- [ ] 通过或有发现的状态
- [ ] 如有发现：严重程度、影响范围、建议处理方式
- [ ] 审计记录（带时间戳、不可篡改）

---

## 8. 决策

**执行者：** 战略决策人（人类）
**位置：** 与协调员私信、`#general`

战略决策人基于汇总输入做出最终决策：
- 协调员的每日报告
- 研究分析师的尽调报告
- 投资工程师的模型产出
- 安全审计员的审计发现

**决策类型：**
- **投/不投：** 执行或放弃某个投资机会
- **再平衡：** 基于模型建议调整持仓
- **上报响应：** 对红色信号事件的即时处理
- **方向调整：** 变更投资论点或策略

---

## 9. 反馈闭环

**执行者：** 全员
**位置：** `#general`、知识库

每个决策周期结束后：
1. 将决策和决策理由记录到知识库
2. 持续追踪决策结果
3. 用实际结果更新模型
4. 定期进行复盘
5. 根据误报/漏报率调整信号分级阈值

```
决策 → 执行 → 追踪 → 复盘 → 知识更新 → 决策
  ↑                                         |
  +-----------------------------------------+
```

## 沟通模式

### 频道

| 频道 | 用途 | 参与者 |
|------|------|--------|
| **#intel** | 信号分级、情报摘要、审计发现 | 协调员、安全审计员 |
| **#research** | 尽调报告、回测结果、模型产出、投资建模 | 研究分析师、协调员、投资工程师 |
| **#general** | 红色信号广播、每日报告、跨团队通知 | 全员 |
| **私信：协调员 ↔ 研究分析师** | 研究任务分派、范围澄清 | 协调员、研究分析师 |
| **私信：协调员 ↔ 投资工程师** | 建模任务分配、数据管道需求 | 协调员、投资工程师 |
| **私信：安全审计员 ↔ 任何人** | 审计问询、合规咨询 | 安全审计员、* |

### 通知流程

```
信号检测 ──→ 协调员在 #intel 中分级
                      │
                 ┌────┴────┐
                 ▼         ▼
           红色信号      黄色信号
                 │         │
                 ▼         ▼
          #general +    每日报告
          私信给        （10:00/22:00）
          战略决策人         │
                           ▼
                   研究分派 ──→ 研究分析师
                           │
                           ▼
                   尽调报告发布到 #research
                           │
                           ▼
                   协调员汇总后呈报战略决策人
```

## 自动化层

| 自动化项 | 负责人 | 实现机制 |
|---------|--------|---------|
| 信号监控 | 协调员 | 持续轮询数据源（市场 API、链上数据、新闻） |
| 信号分级 | 协调员 | 基于严重程度框架的自动分级 |
| 每日报告 | 协调员 | 定时调度器在 10:00 和 22:00 触发 |
| 安全审计 | 安全审计员 | 定时调度器在 03:30 UTC 触发 |
| 研究追踪 | 协调员 | 监控已分派任务状态，跟进逾期项 |
| 模型更新 | 投资工程师 | 由新数据或定时计划触发 |
| 知识库更新 | 全员 | 决策后记录到 Qdrant/SQLite |
