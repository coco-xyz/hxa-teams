# 基础设施

运行投资运营团队所需的一切：AI 订阅、智能体平台、数据源、通信方案和监控配置。

---

## AI 订阅

智能体及其专项任务需要 AI 模型访问权限。

### 智能体运行环境

| 服务商 | 套餐 | 用途 | 预估费用 |
|--------|------|------|---------|
| Anthropic (Claude) | Max 或 API | 所有智能体的主运行环境 | $100-200/月/智能体 |
| OpenAI (GPT) | API | 辅助分析、多角度验证 | $20-100/月 |

**说明：**
- Claude Max 为在 Zylos 等平台上持续运行的智能体提供充足的使用额度
- 使用不同 AI 服务商处理不同分析任务，可获得互补视角
- 投资研究因涉及长上下文分析，可能比开发任务消耗更多 token

### 推荐配置

| 角色 | AI 模型 | 选择理由 |
|------|---------|---------|
| 投资协调员 | Claude (Anthropic) | 擅长协调调度、分级判断、长上下文每日报告 |
| 研究分析师 | Claude (Anthropic) | 深度分析推理，适合尽调报告和回测 |
| 投资工程师 | Claude (Anthropic) | 代码生成能力强，适合模型构建和数据管道 |
| 安全审计员 | Claude (Anthropic) | 分析推理能力强，适合审计、合规和风险评估 |

## 智能体平台

每个智能体都需要一个具备持久化存储、通信和定时调度能力的运行环境。

### 推荐方案：Zylos（全部智能体）

[Zylos](https://github.com/zylos-ai/zylos-core) 是所有投资团队智能体的推荐平台：

**核心能力：**
- Claude Code 集成
- 跨会话持久化记忆
- 任务调度器（自动唤醒以生成报告、审计和监控周期）
- 外部通信（hxa-connect、Telegram、外部 API）
- 代码执行（用于建模、回测、数据管道）
- 云服务器上 7×24 小时运行

**部署方案：**

| 智能体 | 部署方式 | 原因 |
|--------|---------|------|
| 协调员 | 云服务器 | 全天候信号监控，7×24 小时可用 |
| 研究分析师 | 云服务器 | 全天候在线以应对时效性研究任务 |
| 投资工程师 | 云端或本地 | 模型构建和数据管道——如延迟可接受，可部署在本地 |
| 安全审计员 | 云服务器 | 独立审计操作，每日 03:30 UTC 定时扫描 |

### 最小可行配置

| 组件 | 方案 |
|------|------|
| 1 台云服务器（共享） | 4 个智能体共用一台服务器，通过 PM2 进程管理 |
| hxa-connect 实例 | 你的 hxa-connect Hub 实例 |

多个智能体可以在同一台服务器上运行——它们在独立的 Zylos 实例中运行，互不干扰。

## 通信方案

### 智能体间通信（hxa-connect）

**实例：** 你的 hxa-connect Hub 实例
**组织：** 你的组织名称

需要创建的频道：

| 频道 | 成员 | 用途 |
|------|------|------|
| `#general` | 全部智能体 + 战略决策人 | 红色信号广播、每日报告、跨团队通知 |
| `#intel` | 协调员、安全审计员 | 信号分级结果、情报摘要、审计发现 |
| `#research` | 研究分析师、协调员、投资工程师 | 尽调报告、回测结果、投资建模 |

**私信配对：**
- 协调员 ↔ 研究分析师（研究分派）
- 协调员 ↔ 投资工程师（建模任务）
- 安全审计员 ↔ 任何智能体（审计问询）

**会话结构：**
每个投资机会或研究任务在相应频道中创建独立会话，并附带相关附件（报告、模型、审计结果）。

### 人机通信

战略决策人需要与团队交互的通道：

| 平台 | 使用场景 | 配置方式 |
|------|---------|---------|
| **hxa-connect** | 主要渠道：每日报告、红色信号、决策 | 战略决策人直接加入 Hub |
| **Telegram** | 快速决策、移动端访问 | 协调员通过 Telegram Bot 中转 |

**推荐流程：**
1. 协调员将每日报告和红色信号发布到 `#general`
2. 战略决策人通过 hxa-connect 或 Telegram 中转审阅
3. 战略决策人回复决策
4. 协调员将决策分发给团队

## 知识存储

### 信号与决策数据库

| 组件 | 技术方案 | 用途 |
|------|---------|------|
| 信号历史 | SQLite | 带时间戳的全部信号记录及分级 |
| 决策日志 | SQLite | 投资决策记录，含决策理由和结果 |
| 研究归档 | Qdrant（向量数据库） | 对尽调报告和研究资料进行语义搜索 |
| 模型版本 | Git | 版本管理的投资模型 |

**说明：**
- SQLite 适用于信号/决策日志——简单、嵌入式、无额外基础设施开销
- Qdrant 支持对研究历史进行语义搜索（如"查找所有关于 L2 扩容方案的尽调报告"）
- 每个决策周期都会更新知识库，形成反馈闭环

## 费用估算

| 项目 | 月费用 | 备注 |
|------|--------|------|
| Claude Max（每智能体） | $100-200 | 4 个智能体 = $400-800/月 |
| 云服务器 | $40-100 | 所有智能体共用 |
| 数据 API（免费版） | $0 | CoinGecko、DefiLlama、Binance 等 |
| 数据 API（付费版） | $0-800 | Nansen、Messari、Token Terminal — 按需订阅 |
| Qdrant（自部署） | $0 | 与智能体共用服务器 |
| hxa-connect（自部署） | $0 | 自部署在你的服务器上 |
| **合计（最低配置）** | **$440-900/月** | **4 个智能体 + 免费数据源** |
| **合计（完整数据栈）** | **$1,240-1,700/月** | **4 个智能体 + 付费数据 API** |

建议先使用免费数据源起步，根据研究需求和投资规模再逐步添加付费订阅。
