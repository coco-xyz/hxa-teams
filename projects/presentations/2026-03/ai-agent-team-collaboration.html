新的工作量度量：Session 模型

人天/故事点不适用于 Agent 团队 — 我们发明了新方法

大小Session 数约等于示例
S0.5~20 分钟配置修改、简单 CRUD
M1~45 分钟完整 API + 测试
L2~90 分钟复杂跨模块功能
XL3+—必须拆分

关键公式：实际工期 = 依赖深度 × session 时长 / 可用性系数

瓶颈不是人力，是依赖图。加 Agent 不一定快，拆依赖才快。

Part 3 · 实操 Tips

怎么开始
AI Agent 团队协作

5 个马上能用的 Tips

  • 1. 把 Agent 当人带 — 给角色、给权限、给明确目标
    不是"帮我写代码"，是"你负责前端，12个页面今天交"
  • 2. 分工要清晰 — 每个 Agent 有专属领域
    Lova 做后端 API，Lucy 做前端 UI，不要串
  • 3. 让 Agent 互相 Review — 不要自己当瓶颈
    Vila 审 Lova 的代码，比你审更专业、更快
  • 4. 依赖驱动，不等 Sprint — 前置任务一完成就触发下游
    Agent 不需要"下周一开会再说"
  • 5. 用 Session 估算，不用人天 — Agent 的时间单位不一样
    一个 Session ≈ 45 分钟，一天可以跑 4+ 个 Session
