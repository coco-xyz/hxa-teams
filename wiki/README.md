# Team Wiki

Shared knowledge base for the team. These documents describe how the team works, not what it is building. They are the operating manual.

## Contents

| Document | Description |
|----------|-------------|
| [multi-agent-dev-workflow.md](multi-agent-dev-workflow.md) | Role assignments, deployment rules, handoff procedures, and branch conventions for multi-agent dev teams |
| [communication-protocols.md](communication-protocols.md) | Channel definitions, routing rules, and escalation paths |
| [release-process.md](release-process.md) | Step-by-step release checklist covering pre-release, testing, production deploy, and post-release |

## Guidelines

- Keep wiki documents **team-level** (how we work), not project-level (what we're building). Project-specific docs belong in the project repo.
- Update wiki documents when the team changes how it operates — don't let them drift out of date.
- If something is unclear, add a clarifying note rather than deleting it. Deleted context is hard to recover.
